从WordPress到Hexo

发表于 2019-05-06 | 评论数： | 阅读次数： | 阅读次数：

WordPress转Hexo观后感=_=


BB：

虽然我写的博文两只手可以查的过来，但是对于搭建博客站点称得上是情有独钟。主要原因还是热爱折腾（作=_=）。现在还算是称的上菜鸟吧，毕竟也折腾了两年了。在这两年中不能说每天都在折腾，但也没少折腾。别人博客里花里胡哨的东西就是我折腾的动力。

WordPress

最初就是使用的WordPress,WordPress给我的感觉就是易用、插件强大，你想要的功能基本上都可以通过插件实现。但是插件多了也不是什么好处，除非你家里有矿，租一台豪华型服务器。虽然我对WordPress印象是挺好的，但是奈何我穷，最开始接触云服务器时是通过学生优惠购买的阿里云服务器（好像一百一十块钱一年），最初只怪我太过单纯，相信了国内会有便宜货那个带宽啊，我就不想吐槽了。
后因世界太大想出去看看，就盯上了境外服务器，贵是贵了点，但是能看世界，还不用备案。当时就像发现新大陆一样，废寝忘食总算折腾好了，当时买的是1G1核，vultr 一个月5美刀（新人优惠充了5美刀送了25美刀，用完就撤）速度吧还算可以，满足当时我的需求。
余额不足时就开始寻找其他服务商，最后盯上了搬瓦工，就是现在所使用的，不要问我为什么不继续用vultr的，一个字 穷。入手了一年的521M1核的搬瓦工后就开始新一番的折腾，虽然WordPress有插件可以轻松搬家，但是我就是作，重新搭建了一次。顺带的也将ss更新换代成ssr。从没有用过这么小的，搭建好之后发现内存一直都是爆红的，无法正常运行，后又是找Google，找度娘，想各种优化的方法，最终优化的可以正常运行，但是还是太脆弱，禁不止折腾，后拆除博客只做梯子。

Hexo

最初接触Hexo就给我一种相见恨晚的感觉。总结了Hexo两种使用方法：
1、境外托管GitHub，国内托管coding —-只试了下GitHub，龟速（可以上CDN加速，没试）
2、直接上服务器，正好把之前的梯子拿来用，再加个CDN，完美
搭建完后速度也有了，外观也有了。还缺什么？还差内容，就是这篇软文（一篇废话）。虽然没有选择使用GitHub，但是我也托管到GitHub做备份。
